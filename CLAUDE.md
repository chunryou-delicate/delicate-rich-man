# 재무 스크리닝 도구 — 구현 명세서 (CLAUDE.md 초안)

> 폰 기획 대화 결과물. PC에서 Claude Code로 구현 이어갈 때 컨텍스트로 사용.
> 이 문서는 **1단계(스크리너)** 범위만 다룸. 백테스트·자동매매는 후속 단계.

## 0. 프로젝트 목표

- **최종 목적지**: 무인 자동매매 + 실제 수익 (단, 여러 단계 검증 후)
- **현재 단계**: 1단계 — 재무 조건 기반 종목 스크리너 (돈 안 들어감, 후보 걸러내기만)
- **성격**: BDO 재료 스케줄러와 동일 골격 (후보 → 필터 → 스코어링 → 랭킹 → 출력)

### 전체 로드맵 (순서 고정)
1. **스크리너** (현재) — 재무 조건으로 후보 종목 자동 추출
2. **백테스트** — 뽑은 규칙이 과거에 통했는지 검증
3. **모의투자 무인 운용** — KIS 모의계좌, 돈 0원
4. **소액 실전 무인** → 안정 시 증액

## 2. 데이터 소스

| 데이터 | 출처 | 비고 |
|--------|------|------|
| 상장사 목록 + 고유번호(corp_code) | DART OpenAPI | 무료, opendart.fss.or.kr |
| 재무제표 (매출·순이익·자본·부채) | DART OpenAPI | 분기 지연 있음, 캐싱 필수 |
| 시세 / 시가총액 | KIS API | DART엔 주가 없음 → PER·PBR 계산용 |

### 핵심 아키텍처 — "수집"과 "조회"를 분리 (고정 결정)

브라우저(HTML)가 DART API를 **직접 호출하는 것은 불가능**. 이유:
- **CORS 차단**: DART 서버가 브라우저 직접 요청을 거부 (폰·PC 공통, 우회 불가)
- **API 키 노출**: HTML에 키를 박으면 소스보기로 유출
- **데이터 형태**: DART 재무제표는 ZIP/XML 뭉치라 브라우저 파싱 부적합

→ 따라서 **역할을 둘로 나눈다:**

```
[수집기] 파이썬 (PC 또는 클라우드)          [뷰어] HTML (폰·PC 어디서나)
  - DART API 호출 (키는 여기만)               - data.json 을 fetch 해서 읽기
  - 하루 1회 전 종목 재무 수집                - 3층 필터 · 스코어링 · 랭킹
  - 지표 계산 → data.json 저장       ─────▶   - DART를 직접 부르지 않음
  - (선택) KIS 시세 붙여 PER/PBR 보정
```

### 폰에서 실데이터 보는 법 (채택 구조)

수집기가 만든 **data.json 을 웹에 올려두고**, 폰 HTML이 그 URL에서 읽는다:
- 예: GitHub raw, GitHub Pages, 또는 간단한 정적 호스팅에 `data.json` 업로드
- 폰 HTML은 `fetch("<data.json URL>")` 로 실데이터 로드 → **폰에서도 실제 종목이 뜬다**
- **갱신(DART 재호출)은 PC/클라우드 수집기만** 수행, 폰은 결과만 조회
- 즉 "무거운 수집은 서버가, 가벼운 조회는 폰이" — 폰은 항상 최신 JSON만 보면 됨

**뷰어 데이터 소스 스위치**: HTML은 `DATA` 를 두 경로로 받게 설계
1. 개발/오프라인: 내장 더미 배열 (현재 뼈대 상태)
2. 실운영: `fetch(DATA_URL)` 로 원격 data.json 로드
→ URL 하나만 채우면 더미→실데이터 전환. 필터·UI 코드는 그대로.

### 준비물
- **DART OpenAPI 인증키** (무료, 신청 즉시 발급) — opendart.fss.or.kr
- **KIS API 앱키/시크릿** (시세 보완용)
- **data.json 호스팅 위치** (GitHub raw/Pages 등) — 폰 조회용

### data.json 스키마 (수집기 출력 = 뷰어 입력, 계약)

현재 뼈대 HTML의 더미 배열과 **필드명을 일치**시켜, 수집기가 이 형태로 뱉으면
URL만 바꿔 바로 연결된다.

```json
{
  "updated": "2025-07-11T09:00:00+09:00",   // 수집 시각
  "source": "DART+KIS",
  "items": [
    {
      "name": "가온테크",     // 종목명
      "code": "000000",       // 종목코드 (추가)
      "sector": "반도체",     // 섹터
      "cap": 3200,            // 시가총액(억)  ← KIS 시세 기반
      "vol": 180,             // 거래대금(억)  ← KIS
      "roe": 22,              // %             ← DART 재무
      "per": 9,               // 배            ← 시총÷순이익
      "pbr": 1.4,             // 배            ← 시총÷자기자본
      "debt": 45,             // 부채비율 %    ← DART
      "risk": false           // 관리/거래정지/적자 플래그
    }
  ]
}
```

- dummy→실데이터 전환 시 뷰어는 `json.items` 를 `DATA` 로 사용
- `updated` 는 화면 상단에 "기준: N시간 전"으로 표시 → 데이터 신선도 확인용
- 필드 추가되면(예: 관심도 점수) 스키마에 append, 기존 필드는 유지(하위호환)

## 3. 재무 지표 정의 (계산식)

### 성능 (수익성/성장성)
- **ROE** = 순이익 ÷ 자기자본 — 자본 대비 이익 효율. 15%↑ 우량
- **영업이익률** = 영업이익 ÷ 매출
- **매출/이익 성장률** = 전년 대비 증감

### 가격 (밸류에이션)
- **PER** = 시가총액 ÷ 순이익 — 낮을수록 쌈 (업종별 기준 다름)
- **PBR** = 시가총액 ÷ 자기자본 — 1 미만이면 자산보다 싸게 거래 (밸류 트랩 주의)

### 안전 (재무건전성)
- **부채비율** = 부채 ÷ 자기자본 — 100% 이하 선호
- **유동비율** = 유동자산 ÷ 유동부채

## 4. 필터 3층 구조 (핵심 설계)

```
전체 상장사 (국내 코스피+코스닥, 약 2,500개)
  │  ① 자격 필터 — 대량 컷
  ▼
수백 개
  │  ② 품질 필터 — 중간 컷
  ▼
수십 개
  │  ③ 분야 필터 (선택) — 섹터 좁히기
  ▼
최종 후보 (타겟: 20~30개)
```

### ① 자격 필터 (점수 무관, 아예 제외)
- 시가총액 하한 (예: 1,000억 미만 제외) — 소형주 리스크 컷
- 거래대금 하한 (유동성 부족 제외)
- 관리종목 / 거래정지 / 적자기업 제외

### ② 품질 필터 (재무 지표 조건, AND)
예시 프리셋:
- ROE ≥ 15%
- PER ≤ 10 AND PBR ≤ 1.5
- 부채비율 ≤ 100%

#### 품질 필터 확장안 (검증된 퀀트 전략 기반, 실데이터 필요)
> 출처: Greenblatt 마법공식, Piotroski F-Score 등. 전 세계 스크리너가 표준으로 쓰는 지표.
> 공통 원칙: **필터는 매수 신호가 아니라 후보 추리기.** 조합해서 쓸 때 가장 강력.

- **이익수익률 (EBIT / EV)** — 마법공식 핵심. PER보다 정교한 "싼가" 지표
  (EV=시가총액+순부채. 부채까지 반영해 더 공정)
- **자본수익률 (ROC = EBIT / 투하자본)** — 마법공식의 "잘 버는가"
  → 두 지표 각각 전 종목 순위 매겨 **합산 순위**로 랭킹 = 마법공식 방식
- **Piotroski F-Score (0~9)** — 재무 품질/개선 여부. 9개 회계항목 이진 점수
  - 3영역: 수익성 / 재무건전성 / 운영효율
  - 8~9=우량, 0~2=부실. **밸류 트랩 방어 핵심**
  - ⚠ 은행·보험·리츠엔 부적용(재무구조 다름), 2년치 재무 비교 필요
  - F-Score는 품질 지표 → 밸류에이션 지표와 **반드시 함께** (저평가 여부는 못 알려줌)
- **영업현금흐름 > 순이익** — 이익이 실제 현금인지 (분식·회계착시 방어)
- **배당수익률** — 안정성·주주환원 (꾸준한 수익 지향 시 유용)

#### 모멘텀 계열 (더 나중, 관심도 레이어와 연계)
- **상대강도(RS)** — 최근 6개월 주가가 시장 대비 강한가 = "관심도"의 정량 버전
- 결합 목표: "싼 주식(가치) + 재무 개선(F-Score) + 관심 붙음(모멘텀)" 3단

#### 스코어링 방식 (현재 뼈대 → 확장)
- 현재 뼈대: ROE35 + 저PER25 + 저PBR20 + 저부채20 임시 가중식
- 확장 옵션 A(마법공식식): 지표별 순위 합산 → 합산순위 랭킹
- 확장 옵션 B(가중합): 지표별 정규화 후 가중치 곱해 합산 (가중치 UI 조절)
- 실데이터로 어느 방식이 나은지 백테스트(2단계)에서 비교 후 확정

### ③ 분야 필터 (선택 옵션)
- 섹터 선택 (IT만 / 바이오만 등)
- 부가효과: 같은 업종끼리 비교해야 지표가 공정 (PER 기준이 업종마다 다름)
- 기본은 OFF, 특정 분야 집중하고 싶을 때 ON

## 5. UI / 화면 구성

- **단일 파일 HTML** (BDO 스케줄러 방식) + 별도 파이썬 수집기
- 필터 조건을 **슬라이더로 실시간 조절** → 결과 개수 즉시 반영
  - "지금 조건이면 340개... 조여야겠다" 감 잡기용
- 결과 표: 종목별 지표 한눈에, 지표별 정렬(ROE↓, PER↑ 등)
- 결과 저장: CSV / JSON → 백테스트(2단계) 입력으로 재사용

## 6. BDO 스케줄러와의 대응 (참고)

| BDO | 주식 스크리너 |
|-----|--------------|
| 전체 재료 목록 | 전체 상장사 |
| 교환 규칙 필터 | 재무 조건 필터 |
| Phase 2 스코어링 | 지표 가중 스코어링 |
| 랭킹 → 추천 | 랭킹 → 후보 |

**결정적 차이**: BDO는 "규칙 통과 = 이득"(확정). 주식은 "조건 통과 = 후보일 뿐, 오른다는 보장 없음" → 그래서 백테스트(2단계) 검증 관문이 추가로 필요.

## 7. 나중에 붙일 것 (지금은 범위 밖)

### 해외(미국 등) 확장 — 스키마 재활용
- **DART는 국내(코스피+코스닥 약 2,500개) 전용.** 미국주식 재무는 DART에 없음.
- 미국 재무 소스: **SEC EDGAR**(무료, 미국판 DART), Financial Modeling Prep, yfinance 등
- **핵심: 수집기만 하나 더 만들면 됨.** 뷰어·필터·설명·스코어링은 그대로 재활용.
  - `data.json` 스키마가 소스 무관 동일 필드(roe/per/pbr/debt…)라, 미국 수집기가
    같은 형식으로 뱉으면 뷰어는 국내/해외 둘 다 읽음
  - 뷰어에 **"국내 / 미국" 소스 탭**만 추가 (data_kr.json / data_us.json 스위치)
- 해외 추가 변수: 환율, 시차·서머타임, 소수점 거래, 미국 회계기준(GAAP) 차이 → 수집기에서 흡수
- **순서: 국내(DART) 완성 → 미국 수집기 추가.** 국내가 데이터·세금·거래구조 이해가 쉬워 먼저.

### 관심도 레이어 (센티먼트/대체 데이터)
- Google Trends(pytrends), 뉴스량(Tavily/네이버 검색), 네이버 데이터랩 등
- **매수 신호로 직접 쓰지 말 것** (뉴스 뜰 땐 이미 늦음, 노이즈 많음)
- 재무 필터 위에 **분야 필터 보조 지표**로 얹기: "재무 좋은 회사 AND 관심 붙는 섹터"
- **백테스트로 효과 검증 후 채택** (넣었을 때 vs 뺐을 때 비교) ← harness 원칙

### 운용단계 인프라 (실전 무인 시)
- 봇은 **클라우드 서버(VPS)**에 상시 가동 (오라클 무료티어 등) — 집컴 아님
- 폰은 **모니터링/제어용** (상태 대시보드, 긴급정지, 체결 푸시알림)
- 고정 IP(EIP) 증권사 등록 필요

## 8. 스크리너 마무리 결정 (2026-07-12, 실데이터 보고 확정)

1. **재무 데이터 기준 → 연간 확정.** DART 사업보고서(11011), 기본 작년(2025).
   커버리지 최고(ROE 2,584종목), 노이즈 적음. 분기는 `--report half/q1/q3` 로 언제든.
2. **PER/PBR 시세 → KIS 아님, KRX(pykrx)로 확정.** KRX Data Marketplace 무료 로그인.
   KIS는 실전 매매(주문 실행) 단계로 미룸. cap·vol·per·pbr 전부 KRX에서 채움.
3. **스코어링 방식 → 현재 가중합(정규화) 유지, 최종은 백테스트(2단계)에서 확정.**
   값 있는 지표만 가중치 정규화. 마법공식식(순위합)은 EBIT/EV·ROC 추가 수집 필요 → 나중.
4. **후보 개수 → 타겟 20~30, "소형 저평가 포함" 철학.**
   자격필터 완화(기본 시총 300억↓·거래대금 20억↓)로 저PBR 소형주도 후보에 포함.
   대신 거래대금<50억은 카드에 **"저유동" 배지**로 리스크 표시(자동진입 유동성 위험 경고).
   기본 슬라이더 프리셋: 시총300·거래20·ROE12·PER15·PBR2.0·부채130 → 약 24개.

> ⚠️ "소형 저평가 포함"은 수익 여지를 위한 선택이지만 자동진입 시 유동성 리스크가 큼.
> 실제 진입 규칙·체결(3단계)에서 종목당 비중·분할매수로 반드시 방어할 것.

## 9. 안전 원칙 (항상 유지)

- 1단계는 **돈 1원도 안 들어감**. 매수 버튼·주문 없음. 순수 후보 추출.
- 무인 운용은 **최종 목표**지 시작점 아님. 반드시 백테스트 → 모의 → 소액 순.
- 소형주 자동 진입 위험 → 자격 필터(규모·유동성·부실 제외)로 방어.
- 재무제표는 과거, 주가는 미래 → 지표 조건만으로 매수 확정 금지.

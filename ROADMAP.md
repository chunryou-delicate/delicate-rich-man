# 자동매매 시스템 — 전체 로드맵

> 큰 그림 문서. 단계별 "무엇을 / 어떤 방법들이 있나 / 우리 선택"을 한눈에.
> 구현 세부는 각 단계별 CLAUDE.md로 분리. 이 문서는 지도 역할.

핵심 원칙 3가지 (전 단계 공통):
1. **한 번에 다 하지 않는다.** 각 단계는 독립된 스킬 트리. 하나씩 밟는다.
2. **필터·전략은 매수 신호가 아니라 후보·규칙일 뿐.** 검증 전엔 돈 안 넣는다.
3. **무인 운용은 최종 목표지 시작점이 아니다.** 검증→모의→소액→증액 순.

## [1] 스크리닝 — 후보 추리기  ✅ 완료 (2026-07-12)

**목표**: 수천 종목 → 검토 가능한 수십 개 후보로 좁히기 (돈 0원)

**방법 갈래**
- 가치(Value): 마법공식, F-Score, 저PER/PBR ← **우리 선택**
- 퀄리티(Quality): ROE, 영업현금흐름, 부채건전성 ← **우리 선택**
- 성장(Growth): 매출·이익 성장률
- 배당(Dividend): 고배당·주주환원
- 모멘텀(Momentum): 상대강도(RS), 관심도 (나중)

**상태**: ✅ 실데이터까지 완성.
- 수집기: DART(재무 ROE·부채) + KRX/pykrx(시세 cap·vol·per·pbr) → data.json 2,701종목
- 뷰어: 3층 필터 실시간 작동 + 반응형 UI + 저유동 경고 배지
- 결정 확정(CLAUDE.md 8번): 연간 재무 / KRX 시세 / 가중합 스코어링 / "소형 저평가 포함" 철학
**세부**: → CLAUDE.md, collector/README.md 참조

## [2] 백테스트 — 과거에 통했나 검증  ★현재 단계

**목표**: 스크리닝으로 뽑은 규칙을 과거 데이터로 시뮬레이션. "이대로 샀다 팔았으면?"

**방법 갈래**
- 벡터라이즈드: 빠름, 대략 검증용 ← **우리 방식(월 리밸런싱)**
- 이벤트 드리븐: 느리지만 정확(체결·슬리피지 반영)
- 도구: 직접 파이썬 / backtrader / QuantConnect 등

**핵심 위험**: **과최적화(overfitting).** 과거에 딱 맞추면 미래엔 안 통함.
→ 방어: 검증기간 분리(in/out-of-sample), 파라미터 단순하게, 거래비용 꼭 반영

**왜 이 순서**: 여기서 대부분의 환상이 깨지고 전략이 걸러진다. 이거 없이 3단계 이후는 무의미.

**진행 상황** (2026-07-12) — → 세부 backtest/README.md
- 엔진 완성: pykrx 과거 지표(캐싱) → 월 리밸런싱 → CAGR·MDD·샤프 + in/out-of-sample 자동 분할.
- **1차 결과: 저PBR+저PER 순위합 상위20 → 실패.** 2015~2025 CAGR 1.7% vs 코스피 7.3%,
  낙폭 -54%(코스피보다 깊음), 전·후반 모두 언더퍼폼. 순진한 저평가 = 밸류 트랩.
  → 백테스트가 돈 넣기 전에 지는 규칙을 걸러냄 = harness 원칙 작동.
- **품질 필터(F-Score) → ✅ 유효 판정**: PIT·생존편향 검증 완료. F-Score 8항목 +
  가치→품질 2단 엔진 + 사전 판정기준 구현. **전체 2015~2025**에서 가치only(CAGR 1.7%,
  코스피에 -5.7%p) → 가치+품질(**CAGR 8.0%, MDD -54→-38%, 코스피 +0.6%p**),
  사전 확정 기준 4개(CAGR·MDD·in·out) **모두 충족**. "싸다"는 졌고 "싸고+좋다"는 통했다.
  단 초과 +0.6%p는 얇고 MDD -38%는 여전히 큼 → 방어는 진입·청산 단계에서 개선 여지.

**다음**
1. 전 종목 10년 아카이브(collect_pit --full, ~2일) → 어떤 규칙이든 DART 없이 오프라인 실험.
2. 파라미터 민감도(대형주 한정·종목수·주기·비용·F-Score 컷).
4. **백테스트 뷰어(기획)** — engine → backtest.json → HTML(스크리너 골격).
   자산곡선 + 이벤트(전쟁·금리) 오버레이 + 시점 클릭 → 그때 보유종목 드릴다운.
   목적: "미래 예측기"가 아니라 **전략 스트레스 테스트 + 국면별 생존 학습기**.

## [3] 진입 전략 — 언제·얼마 사나

**목표**: 후보를 실제로 어떻게 매수할지

**방법 갈래**
- 정액분할(DCA): 정해진 금액을 나눠 규칙적으로 (타이밍 리스크 분산)
- 변동성 돌파: 특정 조건 돌파 시 진입
- 분할매수: 한 번에 안 사고 나눠서
- 주문 방식: 지정가(가격 지정) vs 시장가(즉시 체결)

**포인트**: 같은 종목도 진입법이 성과를 가른다.

---

## [4] 청산 전략 — 언제 파나

**목표**: 목표 도달·손실 제한을 규칙으로

**방법 갈래**
- 목표가 도달 매도: +X% 달성 시
- 손절(Stop-loss): 고정 -Y% / 트레일링스톱(고점 대비 하락 시)
- 기간 청산: N일/개월 보유 후
- 리밸런싱: 비중 틀어지면 원복

**포인트**: 매수보다 매도가 더 어렵다. 청산 규칙이 실제 손익을 결정.

## [5] 리스크·자금관리 — 얼마씩·분산 (전 단계 관통)

**목표**: 한 번의 실수로 안 죽기. 장기 생존.

**방법 갈래**
- 고정비율: 종목당 자산의 N%만
- 종목/섹터 상한: 한 곳에 몰빵 방지
- 변동성 기반 배분: 변동 큰 종목엔 적게
- 켈리공식: 이론적 최적 베팅 크기(공격적, 주의)
- 전체 손실 한도(circuit breaker): -N% 도달 시 전면 정지

**포인트**: 이게 진짜 핵심. 화려한 전략보다 자금관리가 생존을 가른다.

### [5-A] 위기·폭락장 대응 (판을 엎는 충격)

**전제 — 예측은 불가능하다.** 전쟁·금리충격·시스템 위기는 개인이 못 맞힌다.
위기 때는 **분산이 무력화**된다: 좋은 주식·나쁜 주식 가릴 것 없이 다 같이 빠짐
(상관관계가 1로 수렴). 재무 필터·섹터 분산이 이 순간엔 거의 안 먹힘.
→ 따라서 목표는 "충격을 피하기"(불가능)가 아니라 **"충격에 안 죽기"**.

**대응 3겹 (예측이 아니라 대비)**
1. **자산군 분산** — 주식 안에서 말고 주식 vs 채권 vs 금 vs 현금.
   위기 때 서로 다르게 움직이는 경향(항상은 아님). 스크리너(주식 내부)만으론 방어 안 됨.
2. **시스템 차단기(circuit breaker)** — 자동매매의 핵심 무기.
   "전체 손실 -N% 도달 시 전량 현금화·전면 정지"를 코드에 박음.
   사람은 폭락장에 얼어붙거나 버티다 더 잃지만, 봇은 감정 없이 규칙대로 발 뺌.
3. **손절 / 트레일링스톱** — 개별 종목 -Y% 또는 고점 대비 하락 시 자동 매도.
   하락 초입에 기계적으로 빠지는 장치.

**남는 한계 (정직하게)**
- **갭 하락**: 밤사이 악재로 장 시작이 -20%면 손절가 밑에서 체결됨. 못 막음.
- **전액 방어 불가**: 위기 때 손실 0은 없다. 목표는 "안 죽기"지 "안 잃기"가 아님.
- **차단기의 역설**: 너무 예민하면 잠깐 출렁일 때마다 팔고 반등을 놓침.
  민감도 균형은 백테스트(2단계)로 튜닝.

**핵심 문장**: "수익은 어떻게 버느냐가 아니라 어떻게 안 잃느냐에서 나온다."
예측 대응은 불가능, 피해 최소화 대응은 가능 — 그게 5단계가 사실상 제일 중요한 이유.

## [6] 실행·운영 — 무인 인프라

**목표**: 봇이 24시간 안 꺼지고 돌게

**방법 갈래**
- 서버: 클라우드 VPS(오라클 무료티어 등) — 집컴 아님
- 실행 로직: 토큰 갱신, 호출 한도, 예외처리, 재시도
- 장애 대응: 네트워크 끊김·재부팅에도 안전하게
- 모니터링: 폰=리모컨 (상태 대시보드, 긴급정지, 체결 푸시)
- 보안: 고정 IP(EIP) 증권사 등록, 키 관리

**포인트**: 여기가 "무인 + 실제 수익"이 실현되는 마지막 단계.

# 백테스트 (Backtest) — 로드맵 2단계

스크리닝 규칙을 과거 데이터로 시뮬레이션해 "이대로 샀다 팔았으면?"을 검증한다.
KRX 일자별 지표(pykrx)만 사용 — 재무(DART) 팩터는 point-in-time 수집을 붙인 뒤 추가.

> 배경·안전원칙: [../ROADMAP.md](../ROADMAP.md) [2]. "여기서 대부분의 환상이 깨진다."

## 구조

```
backtest/
  data.py        과거 시세(pykrx) + 디스크 캐싱(.cache/bt) — 규칙 튜닝 반복용
  pit_data.py    과거 재무 point-in-time 조회 (공시일 기준 look-ahead 방어) — F-Score용
  fscore.py      Piotroski F-Score(8항목) 계산, 컷 상수 명시
  engine.py      월별 리밸런싱 루프 (가치 랭킹 → F-Score 필터, 거래비용·생존편향 방어)
  metrics.py     CAGR·MDD·샤프·월간승률
  run.py         실행·리포트 (--compare 로 가치only vs 가치+품질 자동 판정)
  collect_pit.py F-Score 과거재무 사전수집 (느리게·재개형, --full 로 전 종목 아카이브)
  export.py      backtest.json 생성 (뷰어 입력) — 자산곡선·월별보유·이벤트
```
뷰어: `../backtest_viewer.html` (backtest.json 을 fetch) — 자산곡선(품질/가치/코스피),
거시 이벤트(전쟁·금리) 오버레이, 시점 클릭 → 그달 보유종목 드릴다운. 로컬 서버로 연다.
```bash
python -m backtest.export       # backtest.json 갱신
python -m http.server 8000      # → http://127.0.0.1:8000/backtest_viewer.html
```

## 실행

```bash
# 가치only 백테스트
python -m backtest.run                                   # 2015~2025, 상위20
python -m backtest.run --cap-floor 5000 --cost 0.4       # 대형주 한정, 비용 상향

# 품질 필터(F-Score) 비교 + 사전기준 자동 판정
python -m backtest.collect_pit --throttle 0.4            # 먼저 과거재무 캐시 채우기(재개형)
python -m backtest.run --compare                         # 가치only vs 가치+품질 판정

# 전 종목 10년 아카이브 (어떤 규칙이든 오프라인 실험) — 대량, 며칠
python -m backtest.collect_pit --full --throttle 0.4
```
- 첫 실행은 데이터 수집으로 몇 분~, 이후는 `.cache` 캐시라 즉시.
- KRX 로그인 + DART 키(.env) 필요 — [../collector/README.md](../collector/README.md) 참조.
- **DART 대량수집은 rate-limit 주의** — collect_pit 이 throttle·재시도·재개로 대응.

## 규칙 (현재)

- **선정**: 매 리밸런싱일(월 마지막 거래일) PER·PBR 각각 오름차순 순위 → 합산(마법공식식),
  합산순위 낮은(=싼) 상위 N종목. 유니버스: PER>0·PBR>0·시총≥하한·거래대금≥하한.
- **배분**: 동일비중.
- **매매**: 월 1회 전량 리밸런싱.
- **거래비용**: 회전율 × 편도비용 × 양방향 차감(수수료+세금+슬리피지 근사, 기본 0.25%).

## 방어 장치 (백테스트 함정 대응)

- **look-ahead**: 지표는 리밸런싱일 당일 단면만, 수익은 그 이후 기간만.
- **생존편향**: 각 시점의 실제 상장 종목(pykrx 단면)으로 유니버스 구성.
- **과최적화**: 전/후반(in/out-of-sample) 분할을 항상 함께 출력.
- **거래비용**: 반드시 반영(특히 소형주 슬리피지).

## 한계 (정직하게)

- **등락률 근사**: 분할·배당 보정이 완전치 않음(월 단위라 영향 작음).
- **상폐 = 0수익 처리**: 실제 상폐는 거의 총손실 → 성적을 **낙관적으로** 과대평가.
- **배당 무시**: 가치주 배당 미반영 → 전략에 **불리하게** 계산(양방향 편향 상존).

## 품질필터 판정 기준 (사전 확정 2026-07-12 — 결과 보기 전 못 박음)

goalpost 이동/과최적화 방어를 위해 **재실행 전에** 확정한다.
가치only 대비 **가치+품질(F-Score≥컷)** 이 아래를 **모두** 만족해야 "품질 필터 유효":

1. **CAGR 개선** (전체구간)
2. **MDD 개선** (낙폭 덜 깊어짐)
3. **in-sample·out-of-sample 양쪽 다 개선**

→ 한쪽만 좋으면 **과최적화 의심 → 채택 보류.**

- **F-Score 컷오프**: 8점 만점 **≥6** 통과 (9항목 확장 시 **≥7**). `fscore.py` 상수로 명시.
- 신주발행 항목은 v1 제외(8점) — KRX 상장주식수로 9번째 얹으면 컷 ≥7로 상향.

## 품질 필터(F-Score) — ✅ 유효 판정 (2026-07-12)

- **PIT·생존편향 검증 완료** → [pit-verification.md](pit-verification.md).
  공시일(rcept_no)로 look-ahead 방어, 폐지 종목 과거재무 조회 가능(생존편향 OK).
- **구현**: fscore.py(8항목) + pit_data.py(공시일 기준 조회) + engine(가치→품질 2단,
  금융사 자연제외) + run --compare(사전기준 자동 판정).

**전체 2015~2025 결과 (사전 확정 기준 4개 모두 충족 → 유효):**

| | 가치only | 가치+품질 | 코스피 |
|---|---|---|---|
| CAGR | 1.7% | **8.0%** | 7.3% |
| MDD | -54.3% | **-38.2%** | -34.6% |
| 초과CAGR | -5.7%p | **+0.6%p** | — |

- CAGR ✅(1.7→8.0) · MDD ✅(-54→-38) · in-sample ✅(-4.7→0.3) · out-of-sample ✅(8.3→16.1)
- **순진한 저평가(코스피에 -5.7%p)를 F-Score가 구제 → 코스피 상회(+0.6%p).** 밸류 트랩 방어 입증.
- **과신 금지**: 초과수익 +0.6%p는 얇음, MDD -38%는 여전히 큼, 상폐 0처리·배당무시 편향 상존.
- 예비(2020~2023 단독)에선 MDD 악화라 보류였으나 전체 구간에선 방어까지 개선돼 통과.

### ⚠️ 파라미터 민감도 — +0.6%p는 견고하지 않음 (2026-07-12)

top_n·F-Score컷을 바꿔 재실행(캐시)한 결과:

| 설정 | CAGR | MDD | 코스피대비 초과 |
|------|------|-----|------|
| **기준 top20/f6** | 8.0 | -38.2 | **+0.6** |
| top10 / top30 / top40 | 7.0/6.0/5.8 | -40~-43 | -0.3/-1.3/-1.5 |
| f5 / f7 / f8 | 6.2/4.0/1.9 | -41/-45/-56 | -1.2/-3.3/-5.4 |

- **기준 설정만 코스피를 이기고 이웃 파라미터는 전부 짐** → 과최적화 의심(우연한 좋은 자리).
- F-Score컷을 올리면 오히려 붕괴. 모든 설정에서 MDD가 코스피보다 깊음.
- **정직한 재해석**: "F-Score 가치 = 시장 이김"은 과장. 정확히는 **"밸류 트랩은 피하되
  시장과 대략 비슷, 낙폭은 더 큼."** 실전성은 수익 엣지가 아니라 **방어(MDD)** 에서 찾아야 함.
- 단 F-Score가 순진한 가치의 참사(-5.7%p·MDD-54%)를 구제한 것은 견고한 사실.
- **대형주한정(cap하한) 민감도는 아카이브(collect_pit --full) 완료 후** — 다른 종목 필요.

## [3·4] 진입·청산 판정 기준 (사전 확정 2026-07-12 — 결과 보기 전)

민감도가 "수익 엣지 말고 방어"라고 지목 → 손절·트레일링으로 MDD를 줄이되,
**단순 MDD 감소가 아니라 위험 대비 수익이 개선돼야** 유효로 본다:

1. **칼마(Calmar = CAGR / |MDD|) 개선** — 방어가 수익을 깎는 것 이상으로 낙폭을 줄였는가.
2. **샤프 개선**.
3. 손절이 반등을 잘라 **CAGR을 크게 깎으면 실패**(방어값이 없음).
4. **처음부터 민감도까지** — 손절 -%·트레일링 폭을 그리드로 돌려 **여러 설정에서 유지**돼야
   유효. 또 한 자리에서만 좋으면(과최적화) 무효.

기준선 = 손절 없는 base(top20/f6). 그리드가 칼마·샤프를 **일관되게** 올리면 채택.

### 결과 1차 (종가 낙관 모델) — ⚠️ 아티팩트, 아래 갭 재검증으로 반증됨(무효)

> 이 표는 "손절선에서 정확히 체결" 가정의 낙관치. 체결 모델을 고치고 갭을 반영하니
> 개선이 사라짐(다음 절). 기록만 남기고 **채택하지 않음.**

| 설정 | CAGR | MDD | 칼마 | 샤프 |
|------|------|-----|------|------|
| base(무손절) | 7.8 | -35.9 | 0.22 | 0.45 |
| 고정 -10% | 11.1 | -27.4 | 0.40 | 0.64 |
| 고정 -15% | 9.2 | -33.4 | 0.28 | 0.52 |

- 처음엔 "고정 -10% 압권(칼마 2배)"으로 보였으나 → **낙관 아티팩트**(체결 가정 탓).

### 갭/장중 재검증 판정 기준 (사전 확정 2026-07-12 — 결과 보기 전)

일별 종가 기준 결과(고정-10% 칼마0.40)는 **갭·장중을 무시한 낙관 상한선**. 실체 확인:
- **모델**: 매수=d0 종가. 이후 일별로 판정 — 갭다운으로 손절선 관통 시 **시가(더 나쁜 가격) 체결**,
  장중 저가가 손절선 터치 시 손절가 체결. (종가만 볼 때보다 보수적)
- **판정**: 갭 반영 후에도 손절이 base(무손절) 대비 **칼마·샤프를 개선**하고,
  **"타이트할수록 좋은" 경향이 유지**되면 → 견고(유효). 개선이 사라지거나 한 폭에서만이면 → 낙관치는 허상, 다운그레이드.
- 낙관치(종가) 대비 갭 반영치가 얼마나 깎이는지 나란히 기록. 실제 성적 = 그 사이.

### 갭/장중 재검증 결과 (exit_sim.py, 2026-07-12) — ❌ 손절 무효 (반증)

| 설정 | close(낙관) CAGR/칼마 | **gap(현실)** CAGR/MDD/칼마/샤프 | 판정 |
|------|------|------|------|
| base(무손절) | 7.8 / 0.22 | 7.8 / -35.9 / 0.22 / 0.45 | — |
| 고정 -10% | 7.3 / 0.22 | **3.4 / -36.9 / 0.09 / 0.26** | ❌ |
| 고정 -15% | 7.2 / 0.19 | 2.8 / -42.4 / 0.07 / 0.23 | ❌ |
| 고정 -20% | 6.7 / 0.16 | 2.0 / -47.2 / 0.04 / 0.20 | ❌ |
| 트레일 15/20/25% | 7.9/7.6/6.9 | 3.1/2.8/2.5, 칼마 0.08/0.06/0.05 | ❌ |

- **gap 기준 0/6 개선.** 고정-10% 칼마 0.40(낙관)→**0.09(현실)**, 샤프 0.64→0.26.
- **체결 모델만 고쳐도(종가 실체결) 칼마가 base와 동일(0.22) — 낙관치는 "-10%에 정확히
  체결" 가정의 산물이었음.** 갭까지 반영하니 **CAGR 반토막(7.8→3.4), MDD는 오히려 더 깊음**.
- 원인: 갭다운이 손절선을 뚫고 더 나쁜 가격 체결 + 변동성 whipsaw로 반등 놓침.
- **결론: 단순 고정/트레일링 손절은 이 전략에 무효, 오히려 해롭다.** 1차 "유효"(커밋 2ae7110)는
  낙관 모델 아티팩트로 **반증·철회**. 방어는 다른 방법(자산배분·현금비중·차단기 등, 로드맵 5)에서.
- 규율 성과: 갭 재검증이 허상을 걸러냄. 낙관 백테스트를 실전으로 착각 안 함.
- **모델 검증(버그 아님)**: 실종목 추적 + 손절빈도 확인. gap이 close보다 자주 발동
  (26% vs 21%, 장중 스톱 현실 반영), whipsaw(무손절이면 +) gap 8%>close 4%.
  -10% 손절이 월 보유의 21~26%를 발동 → 손실누적+반등놓침이 꼬리보호 웃돎. 결론 확증.

## [5] 포트폴리오 방어 판정 기준 (사전 확정 2026-07-12 — 결과 보기 전)

**🎯 목표 재정의**: 개별 손절이 갭에 취약해 실패 → 이제 목표를 **"시장 이기기"가 아니라
"시장 수익 유지하며 MDD 줄이기"**로 명시 변경. 얇은 초과수익 사냥은 3번(가치/품질/손절) 벽에 부딪힘.

**가설**: 전체 포트폴리오 레벨 방어(월 단위·전체 드로다운 기준)는 개별종목 손절과 달리
장중 갭에 덜 취약할 것. (개별 손절은 종목별 장중 저가에 21~26% 발동돼 whipsaw였음)

**검증 대상**(둘, 갭 무관 — 월 단위 신호):
- **정적 현금비중**: 상시 c% 현금 + (1-c)% 전략. (트리비얼 null — 그냥 덜 담기)
- **차단기(circuit breaker)**: 전략 누적이 고점 대비 -N% 도달 → 다음달 현금화, 회복 시 재진입.

**판정**(사전 확정):
1. **칼마·샤프 개선** (목표=MDD를 CAGR보다 더 줄임).
2. **차단기가 정적 현금비중(null)보다 나아야** — 아니면 "그냥 덜 담기"와 다를 게 없음.
3. **민감도 그리드**(N·c 여러 값)에서 유지. 한 자리만 좋으면 과최적화 기각.
4. CAGR이 시장(코스피 7.3%) 근처로 유지되는지 함께 확인(목표=시장 수익 유지).

| [코스피] | 7.3 | -34.6 | 0.21 | 0.48 | — |

| 정적현금 20% | 6.7 | -31.1 | 0.22 | 0.45 | 🔶 목표충족(트리비얼) |

- **❌ 차단기(반응형 드로다운 현금화) 실패**: CAGR 8.0→~2%로 박살, MDD는 못 줄임.
  **원인은 갭 아님**(월 단위라 갭 무관) → **반응형 타이밍 본질**. -N% "도달 후" 파니 손실은 이미 났고,
  현금화 후 **반등 놓침** + 회복 후 재진입 고점 매수 = 개별 손절과 **똑같은 whipsaw**.
  → "포트폴리오 레벨은 갭에 덜 취약" 가설 **반증**(갭이 아니라 타이밍이 문제).
- **🔶 정적현금 20%가 재정의 목표 충족**: CAGR 6.7%(≈시장), MDD -31%(<시장 -35%). 단 칼마는
  base와 동일 — **"기술"이 아니라 그냥 덜 담기**(위험/수익 선을 따라 내려갔을 뿐, 무위험 개선 없음).
- **세 번째 벽/교훈**: 반응형·타이밍 방어(손절이든 차단기든)는 평균회귀 전략을 whipsaw로 망침.
  MDD를 안정적으로 줄이는 정직한 도구는 **"덜 담기(현금비중)"뿐** — 칼마↑는 공짜로 안 나옴.
- **다음 후보(미결정)**: 반응형 말고 **선제적** 방어(추세필터/이동평균 국면), 또는 자산배분
  (주식 외 채권·금). 단 이건 새 데이터·설계 → 박사님 결정.

## E0: 실행 현실화 (판정기준 사전 확정 2026-07-12 — 결과 보기 전)

> 실험 규율은 [../EXPERIMENT_PLAN.md](../EXPERIMENT_PLAN.md). E0는 전략 실험이 아니라 **토대 작업.**

- **가설**: 현 엔진은 look-ahead(선정 d0 종가·매수 d0 시가)+슬리피지 무시로 성적이 부풀려짐.
  이를 걷어내면 성적은 보수적으로 나빠져야 정상.
- **작업**: ①look-ahead 수정(매수 d0 종가) ②슬리피지 반영 ③실행 슬롯화(execute).
- **판정 (두 갈래 — 일반 실험과 다름)**:
  - **(A) 인프라 정확성**: 성공 = **성적이 보수적으로 나빠짐.** look-ahead·슬리피지 제거했으니
    하락이 정상이고 그게 성공. **안 나빠지거나 좋아지면 → 현실화 덜 됨/버그 의심.**
    성적 하락을 "실패"로 오해 금지. 거짓을 걷어내는 작업.
  - **(B) 부수효과**: 현실 토대에서 기존 판정 재확인 — 죽은 손절·차단기 그대로 죽는지,
    F-Score 취약성 재확인되는지. (죽어야 정상)
- **단계별 기록**: 현상태 → look-ahead만 수정 → +슬리피지, 각 단계 따로 찍어 **어느 비현실이
  얼마나 부풀렸나 분해.** 한 번에 다 바꾸고 "나빠졌네"보다 학습됨.

### E0 결과 (execution.py, 2026-07-12) — (A)✅ (B)✅, 그리고 큰 발견

| 단계 | CAGR | MDD | 칼마 | 샤프 |
|------|------|-----|------|------|
| [코스피] | 7.3 | -34.6 | 0.21 | 0.48 |
| 현상태(엔진) | 8.0 | -38.2 | 0.21 | 0.45 |
| +look-ahead수정 | 7.8 | -35.9 | 0.22 | 0.45 |
| **++슬리피지0.2% (현실 토대)** | **6.0** | **-38.2** | **0.16** | **0.38** |

- **분해**: look-ahead +0.2%p(미미, 하루 타이밍), **슬리피지 +1.8%p(주범)** — 월 37% 회전이라 마찰이 매년 ~1.8% 잠식.
- **(A) ✅**: 현실화 후 칼마 0.21→0.16, 보수적으로 나빠짐 = 정상(버그 아님).
- **(B) ✅**: 현실 토대에서도 차단기(칼마 0.01~0.02)·손절 그대로 죽음, 정적현금 여전히 트리비얼.
- **🔴 큰 발견**: **현실 토대에선 전략이 코스피에 모든 지표로 짐**(CAGR 6.0<7.3·MDD -38<-35·칼마 0.16<0.21·샤프 0.38<0.48). 앞선 "F-Score +0.6%p 시장 상회"는 **슬리피지-무시 인플레**였음.
- **함의**: 고회전(월37%)·슬리피지가 최대 드래그. 다음 지렛대 후보 = **회전율 감축**(리밸 주기↑ 등).
  목표 C(시장수익 유지+MDD↓)를 현실 토대는 아직 **둘 다 못 함** → 방어가 아니라 비용/회전 구조가 문제.

## I2: 추세필터 (본진) — 판정기준 사전 확정 (2026-07-14, 결과 보기 전)

- **가설 H1**: 지수 ≥ N개월 이평(SMA)이면 주식, 아니면 현금(+기준금리). 개별종목과 달리
  인덱스는 whipsaw 감내 + 2008 회피 → MDD 구조적 감소.
- **규칙**: 신호 = t월말 지수 vs N개월 SMA(±버퍼밴드 히스테리시스) → **t+1월** 주식/현금 체결.
  주식월 = 가격+배당-보유드래그, 현금월 = 기준금리. 상태 전환 시 스위치 0.1%.
- **그리드(12)**: N = 5/8/10/12개월 × 버퍼 = 0/±1/±3%.
- **개별 config PASS**: ①전체 §1(CAGR≥6.6·MDD낙폭≤29·칼마≥0.24) ②**검증(2013-19 박스피)
  칼마 ≥ 0.13**(B&H, whipsaw 견딤) ③연 매매 ≤ 4회. 셋 다여야 통과.
- **그리드 회색지대 규칙(사전 확정 — 판정 시비 차단)**:
  **≥7/12 통과 = 유효 · 딱 6/12 = 부분유효(기록+원인진단) · ≤5/12 = 기각.**
- **기여 분해(필수, 메커니즘 결론)**:
  ① 훈련 MDD 개선이 **2008 회피**에서 왔나 (2008 구간 전략 낙폭 vs B&H -47.5%).
  ② 검증 **whipsaw가 연 몇 %p** 깎나 (검증 전략CAGR vs B&H 2.7%, 스위치 횟수).
  통과든 탈락이든 **메커니즘 수준 결론**이 나와야 다음 수 결정.

## I3: 변동성 타겟팅 — 판정기준 사전 확정 (2026-07-14, 결과 보기 전)

- **가설 H2**: 목표변동성 T / 실현변동성(L일)으로 주식비중 **0~100% 연속**. on/off가 아니라
  부드러워 박스피 whipsaw↓, 위기 초입(변동성 급등)엔 자동 감속.
- **규칙**: 월말 실현변동성(최근 L거래일 일수익 std × √252) → 비중 = clip(T/vol, 0, 1)
  → **t+1** 체결. 비중변화 × 0.1% 비용.
- **그리드(6)**: T = 10/12/15% × L = 20/60일.
- **개별 PASS**: §1 동일(전체 CAGR≥6.6·MDD낙폭≤29·칼마≥0.24) + 검증(박스피) 칼마≥0.13
  + 연 회전(매매환산) ≤ 4.
- **회색지대 규칙(6개, 사전확정)**: **≥4/6 유효 · 3/6 부분유효 · ≤2 기각.**
- **⚠ I3 고유 함정**: 실현변동성은 **후행** → 급락 초입 감속 지연 가능. **위기별 감속 타이밍
  (며칠/몇달 만에 비중↓) 분해**. 2008(서서히) vs 2020(급락)에서 다르게 나올 것.
- **기여 분해(I2와 같은 축)**: ①위기 방어(2008·2020 낙폭 vs B&H) ②박스피 출혈(검증 CAGR vs B&H)
  ③회전율/비용. + **I2 최선 config와 나란히 비교표.**
- **목적**: I3가 '박스피 버티고 위기방어 약함'이면 → I2+I3 조합(I5) 근거.

## I5: I2+I3 조합 — 판정기준 사전 확정 (2026-07-14, 결과 보기 전)

- **구조**: 곱 — 비중 = 추세신호(0/1) × 변동성비중(0~1). 추세 이탈=즉시 0(빠른 방어),
  추세 안=변동성 스케일(횡보 완화). 신호 t월말 → t+1 체결.
- **① 파라미터 폭발 금지**: I2·I3 **검증된 대표값만** 조합(새 탐색 없음). 그리드 4개:
  추세 {N8/±0, N10/±1%} × 변동성 {T12/L60, T15/L60}. 재튜닝 아님 = 결합방식 검증. 다중검정 카운터 계속.
- **② 재입장 비용 분해**: 곱 구조는 추세 0→1 복귀 시 비중 점프. 박스피 추세 깜빡임(I2 22회)마다
  점프하면 I3 부드러움 무력화 → **재입장 횟수·비용 따로 측정**(I3 회전 0.7 대비).
- **③ 이중 관문(사전 수치화)**: §1 전부(CAGR≥6.6·MDD낙폭≤29·칼마≥0.24·검칼마≥0.13·연회전≤4)
  **+ 훈련 MDD ≥ -30%**(I2 ~-27% 근접, 위기방어 보존) **+ 검증 CAGR ≥ 1.3%**(I3 1.8% 근접, 횡보 보존).
  **부모(I2/I3) 어느 쪽보다 퇴보하면 조합 실패.**
- **회색지대(4개)**: ≥3/4 유효 · 2/4 부분유효 · ≤1 기각.

## I5b: 블렌드 조합 — 사전등록 + 갈래 (2026-07-14, 결과 보기 전)

- **구조**: 합산 블렌드 — 비중 = a·추세(0/1) + (1-a)·변동성비중. **a=0.5 고정**(필요시 {0.3/0.5/0.7}).
  곱의 재입장 점프 제거, 추세=0에도 (1-a)·vol 남음. 관문은 **원래 그대로**(수정 없음).
- **① 새 튜닝 차원 금지**: 부모는 검증된 대표값(추세{8/±0,10/±1}×vol{T12,T15}/L60), 블렌드 고정. 다중검정 계속.
- **② 예측(명시)**: 블렌드는 추세=0에도 vol 절반 남아 **위기 방어 희석** → 이번엔 **train MDD≤29
  쪽에서 죽을 수 있음.** 그러면 '상충 프론티어 보존'의 증거로 기록(부모 강점 못 합침).
- **③ 재입장 점프 크기·비용** I5와 같은 축 분해.
- **갈래(사전등록)**:
  - **(a) 어느 config든 §1 5/5 통과** → walk-forward(롤링 창) → 통과 시 **봉인(테스트 2020-25) 1회.**
  - **(b) 5/5 없음(어느 쪽이든 실패)** → **3국면 한국 campaign '통과 후보 없음'으로 원래 규칙대로 종결.**
    봉인은 **안 태움**(통과 후보 전용). LESSONS에 기록: 관문 설계 교훈("박스피 관문은 보험이
    공짜이길 요구 — 강건성(출혈 상한)과 무사고 구간 초과수익을 혼동") + **최선 config 트레이드오프
    프로파일(CAGR/MDD/박스드래그 %p) 숫자.** 관문 수정 소급 금지 → 다음 campaign(미장 I7)에서 새 사전등록.

- **예측 ② 빗나감**: train MDD -23%로 통과(희석됐으나 관문 안). 2008 -12%(I2 -7보다 약하나 방어).
- **단 1/4만 통과** → 요행 가능(회색지대 유효 ≥3/4 미달). **갈래 (a) → walk-forward 견고성 검증 필수.**

### I5b walk-forward (i5b_walkforward.py, 2026-07-14) — ⚠️ 견고성 부족

- **① 테스트 제외(2000-19)**: CAGR **6.5%** — 관문 6.6에 **0.1%p 미달**(전체 5/5는 테스트 캐리).
  MDD -23·칼마 0.28·검칼마 0.15는 통과.
- **② 롤링 5년 창(16개)**: 위기 방어 견고(2000-12 전부 MDD 대폭↓), 단 **박스피(2009-16) 출혈**
  (2011-15 전략 -3.5 vs B&H +0.2, MDD도 안 나음). MDD감소+수익유지 **둘 다 9/16**(일관 70% 미달).
- **정체 확정**: **견고한 위기 헤지 + 박스피 비용.** 위기엔 일관되게 MDD 반토막, 횡보엔 수익만 깎음.
- **갈래 판정**: walk-forward 미확인(테스트 캐리 + 박스피 비일관) → **(a) 조건 미충족 → (b) 종결**.
  봉인 안 태움. 단 0.1%p라 애매 → 박사님 최종 판단(회색지대 기록).

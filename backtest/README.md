# 백테스트 (Backtest) — 로드맵 2단계

스크리닝 규칙을 과거 데이터로 시뮬레이션해 "이대로 샀다 팔았으면?"을 검증한다.
KRX 일자별 지표(pykrx)만 사용 — 재무(DART) 팩터는 point-in-time 수집을 붙인 뒤 추가.

> 배경·안전원칙: [../ROADMAP.md](../ROADMAP.md) [2]. "여기서 대부분의 환상이 깨진다."

## 구조

```
backtest/
  data.py        과거 시세(pykrx) + 디스크 캐싱(.cache/bt) — 규칙 튜닝 반복용
  pit_data.py    과거 재무 point-in-time 조회 (공시일 기준 look-ahead 방어) — F-Score용
  fscore.py      Piotroski F-Score(8항목) 계산, 컷 상수 명시
  engine.py      월별 리밸런싱 루프 (가치 랭킹 → F-Score 필터, 거래비용·생존편향 방어)
  metrics.py     CAGR·MDD·샤프·월간승률
  run.py         실행·리포트 (--compare 로 가치only vs 가치+품질 자동 판정)
  collect_pit.py F-Score 과거재무 사전수집 (느리게·재개형, --full 로 전 종목 아카이브)
  export.py      backtest.json 생성 (뷰어 입력) — 자산곡선·월별보유·이벤트
```
뷰어: `../backtest_viewer.html` (backtest.json 을 fetch) — 자산곡선(품질/가치/코스피),
거시 이벤트(전쟁·금리) 오버레이, 시점 클릭 → 그달 보유종목 드릴다운. 로컬 서버로 연다.
```bash
python -m backtest.export       # backtest.json 갱신
python -m http.server 8000      # → http://127.0.0.1:8000/backtest_viewer.html
```

## 실행

```bash
# 가치only 백테스트
python -m backtest.run                                   # 2015~2025, 상위20
python -m backtest.run --cap-floor 5000 --cost 0.4       # 대형주 한정, 비용 상향

# 품질 필터(F-Score) 비교 + 사전기준 자동 판정
python -m backtest.collect_pit --throttle 0.4            # 먼저 과거재무 캐시 채우기(재개형)
python -m backtest.run --compare                         # 가치only vs 가치+품질 판정

# 전 종목 10년 아카이브 (어떤 규칙이든 오프라인 실험) — 대량, 며칠
python -m backtest.collect_pit --full --throttle 0.4
```
- 첫 실행은 데이터 수집으로 몇 분~, 이후는 `.cache` 캐시라 즉시.
- KRX 로그인 + DART 키(.env) 필요 — [../collector/README.md](../collector/README.md) 참조.
- **DART 대량수집은 rate-limit 주의** — collect_pit 이 throttle·재시도·재개로 대응.

## 규칙 (현재)

- **선정**: 매 리밸런싱일(월 마지막 거래일) PER·PBR 각각 오름차순 순위 → 합산(마법공식식),
  합산순위 낮은(=싼) 상위 N종목. 유니버스: PER>0·PBR>0·시총≥하한·거래대금≥하한.
- **배분**: 동일비중.
- **매매**: 월 1회 전량 리밸런싱.
- **거래비용**: 회전율 × 편도비용 × 양방향 차감(수수료+세금+슬리피지 근사, 기본 0.25%).

## 방어 장치 (백테스트 함정 대응)

- **look-ahead**: 지표는 리밸런싱일 당일 단면만, 수익은 그 이후 기간만.
- **생존편향**: 각 시점의 실제 상장 종목(pykrx 단면)으로 유니버스 구성.
- **과최적화**: 전/후반(in/out-of-sample) 분할을 항상 함께 출력.
- **거래비용**: 반드시 반영(특히 소형주 슬리피지).

## 한계 (정직하게)

- **등락률 근사**: 분할·배당 보정이 완전치 않음(월 단위라 영향 작음).
- **상폐 = 0수익 처리**: 실제 상폐는 거의 총손실 → 성적을 **낙관적으로** 과대평가.
- **배당 무시**: 가치주 배당 미반영 → 전략에 **불리하게** 계산(양방향 편향 상존).

| | 전략 | 코스피 |
|---|---|---|
| 누적수익 (2015~2025) | +19.6% | +116.2% |
| CAGR | 1.7% | 7.3% |
| MDD | **-54.3%** | -34.6% |
| 초과CAGR | **-5.7%p** | — |

- **전반·후반 모두 코스피에 언더퍼폼.** 덜 벌고(1/6) 더 아팠다(낙폭 더 큼).
- **결론: 순진한 저평가 규칙은 실패.** 밸류 트랩(일회성 이익·사양산업)을 걸러낼
  **품질 필터(F-Score·ROE·영업현금흐름)가 없어서** — CLAUDE.md가 예고한 지점.
- 백테스트가 **돈 넣기 전에** 지는 규칙임을 밝혀냄 = harness 원칙 작동.

## 품질필터 판정 기준 (사전 확정 2026-07-12 — 결과 보기 전 못 박음)

goalpost 이동/과최적화 방어를 위해 **재실행 전에** 확정한다.
가치only 대비 **가치+품질(F-Score≥컷)** 이 아래를 **모두** 만족해야 "품질 필터 유효":

1. **CAGR 개선** (전체구간)
2. **MDD 개선** (낙폭 덜 깊어짐)
3. **in-sample·out-of-sample 양쪽 다 개선**

→ 한쪽만 좋으면 **과최적화 의심 → 채택 보류.**

- **F-Score 컷오프**: 8점 만점 **≥6** 통과 (9항목 확장 시 **≥7**). `fscore.py` 상수로 명시.
- 신주발행 항목은 v1 제외(8점) — KRX 상장주식수로 9번째 얹으면 컷 ≥7로 상향.

## 품질 필터(F-Score) — ✅ 유효 판정 (2026-07-12)

- **PIT·생존편향 검증 완료** → [pit-verification.md](pit-verification.md).
  공시일(rcept_no)로 look-ahead 방어, 폐지 종목 과거재무 조회 가능(생존편향 OK).
- **구현**: fscore.py(8항목) + pit_data.py(공시일 기준 조회) + engine(가치→품질 2단,
  금융사 자연제외) + run --compare(사전기준 자동 판정).

**전체 2015~2025 결과 (사전 확정 기준 4개 모두 충족 → 유효):**

| | 가치only | 가치+품질 | 코스피 |
|---|---|---|---|
| CAGR | 1.7% | **8.0%** | 7.3% |
| MDD | -54.3% | **-38.2%** | -34.6% |
| 초과CAGR | -5.7%p | **+0.6%p** | — |

- CAGR ✅(1.7→8.0) · MDD ✅(-54→-38) · in-sample ✅(-4.7→0.3) · out-of-sample ✅(8.3→16.1)
- **순진한 저평가(코스피에 -5.7%p)를 F-Score가 구제 → 코스피 상회(+0.6%p).** 밸류 트랩 방어 입증.
- **과신 금지**: 초과수익 +0.6%p는 얇음, MDD -38%는 여전히 큼, 상폐 0처리·배당무시 편향 상존.
- 예비(2020~2023 단독)에선 MDD 악화라 보류였으나 전체 구간에선 방어까지 개선돼 통과.

### ⚠️ 파라미터 민감도 — +0.6%p는 견고하지 않음 (2026-07-12)

top_n·F-Score컷을 바꿔 재실행(캐시)한 결과:

| 설정 | CAGR | MDD | 코스피대비 초과 |
|------|------|-----|------|
| **기준 top20/f6** | 8.0 | -38.2 | **+0.6** |
| top10 / top30 / top40 | 7.0/6.0/5.8 | -40~-43 | -0.3/-1.3/-1.5 |
| f5 / f7 / f8 | 6.2/4.0/1.9 | -41/-45/-56 | -1.2/-3.3/-5.4 |

- **기준 설정만 코스피를 이기고 이웃 파라미터는 전부 짐** → 과최적화 의심(우연한 좋은 자리).
- F-Score컷을 올리면 오히려 붕괴. 모든 설정에서 MDD가 코스피보다 깊음.
- **정직한 재해석**: "F-Score 가치 = 시장 이김"은 과장. 정확히는 **"밸류 트랩은 피하되
  시장과 대략 비슷, 낙폭은 더 큼."** 실전성은 수익 엣지가 아니라 **방어(MDD)** 에서 찾아야 함.
- 단 F-Score가 순진한 가치의 참사(-5.7%p·MDD-54%)를 구제한 것은 견고한 사실.
- **대형주한정(cap하한) 민감도는 아카이브(collect_pit --full) 완료 후** — 다른 종목 필요.

## [3·4] 진입·청산 판정 기준 (사전 확정 2026-07-12 — 결과 보기 전)

민감도가 "수익 엣지 말고 방어"라고 지목 → 손절·트레일링으로 MDD를 줄이되,
**단순 MDD 감소가 아니라 위험 대비 수익이 개선돼야** 유효로 본다:

1. **칼마(Calmar = CAGR / |MDD|) 개선** — 방어가 수익을 깎는 것 이상으로 낙폭을 줄였는가.
2. **샤프 개선**.
3. 손절이 반등을 잘라 **CAGR을 크게 깎으면 실패**(방어값이 없음).
4. **처음부터 민감도까지** — 손절 -%·트레일링 폭을 그리드로 돌려 **여러 설정에서 유지**돼야
   유효. 또 한 자리에서만 좋으면(과최적화) 무효.

기준선 = 손절 없는 base(top20/f6). 그리드가 칼마·샤프를 **일관되게** 올리면 채택.

### 결과 1차 (종가 낙관 모델) — ⚠️ 아티팩트, 아래 갭 재검증으로 반증됨(무효)

> 이 표는 "손절선에서 정확히 체결" 가정의 낙관치. 체결 모델을 고치고 갭을 반영하니
> 개선이 사라짐(다음 절). 기록만 남기고 **채택하지 않음.**

| 설정 | CAGR | MDD | 칼마 | 샤프 |
|------|------|-----|------|------|
| base(무손절) | 7.8 | -35.9 | 0.22 | 0.45 |
| 고정 -10% | 11.1 | -27.4 | 0.40 | 0.64 |
| 고정 -15% | 9.2 | -33.4 | 0.28 | 0.52 |

- 처음엔 "고정 -10% 압권(칼마 2배)"으로 보였으나 → **낙관 아티팩트**(체결 가정 탓).

### 갭/장중 재검증 판정 기준 (사전 확정 2026-07-12 — 결과 보기 전)

일별 종가 기준 결과(고정-10% 칼마0.40)는 **갭·장중을 무시한 낙관 상한선**. 실체 확인:
- **모델**: 매수=d0 종가. 이후 일별로 판정 — 갭다운으로 손절선 관통 시 **시가(더 나쁜 가격) 체결**,
  장중 저가가 손절선 터치 시 손절가 체결. (종가만 볼 때보다 보수적)
- **판정**: 갭 반영 후에도 손절이 base(무손절) 대비 **칼마·샤프를 개선**하고,
  **"타이트할수록 좋은" 경향이 유지**되면 → 견고(유효). 개선이 사라지거나 한 폭에서만이면 → 낙관치는 허상, 다운그레이드.
- 낙관치(종가) 대비 갭 반영치가 얼마나 깎이는지 나란히 기록. 실제 성적 = 그 사이.

### 갭/장중 재검증 결과 (exit_sim.py, 2026-07-12) — ❌ 손절 무효 (반증)

| 설정 | close(낙관) CAGR/칼마 | **gap(현실)** CAGR/MDD/칼마/샤프 | 판정 |
|------|------|------|------|
| base(무손절) | 7.8 / 0.22 | 7.8 / -35.9 / 0.22 / 0.45 | — |
| 고정 -10% | 7.3 / 0.22 | **3.4 / -36.9 / 0.09 / 0.26** | ❌ |
| 고정 -15% | 7.2 / 0.19 | 2.8 / -42.4 / 0.07 / 0.23 | ❌ |
| 고정 -20% | 6.7 / 0.16 | 2.0 / -47.2 / 0.04 / 0.20 | ❌ |
| 트레일 15/20/25% | 7.9/7.6/6.9 | 3.1/2.8/2.5, 칼마 0.08/0.06/0.05 | ❌ |

- **gap 기준 0/6 개선.** 고정-10% 칼마 0.40(낙관)→**0.09(현실)**, 샤프 0.64→0.26.
- **체결 모델만 고쳐도(종가 실체결) 칼마가 base와 동일(0.22) — 낙관치는 "-10%에 정확히
  체결" 가정의 산물이었음.** 갭까지 반영하니 **CAGR 반토막(7.8→3.4), MDD는 오히려 더 깊음**.
- 원인: 갭다운이 손절선을 뚫고 더 나쁜 가격 체결 + 변동성 whipsaw로 반등 놓침.
- **결론: 단순 고정/트레일링 손절은 이 전략에 무효, 오히려 해롭다.** 1차 "유효"(커밋 2ae7110)는
  낙관 모델 아티팩트로 **반증·철회**. 방어는 다른 방법(자산배분·현금비중·차단기 등, 로드맵 5)에서.
- 규율 성과: 갭 재검증이 허상을 걸러냄. 낙관 백테스트를 실전으로 착각 안 함.
- **모델 검증(버그 아님)**: 실종목 추적 + 손절빈도 확인. gap이 close보다 자주 발동
  (26% vs 21%, 장중 스톱 현실 반영), whipsaw(무손절이면 +) gap 8%>close 4%.
  -10% 손절이 월 보유의 21~26%를 발동 → 손실누적+반등놓침이 꼬리보호 웃돎. 결론 확증.

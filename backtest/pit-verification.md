# PIT·생존편향 검증 노트 (2026-07-12)

품질 필터(F-Score)를 과거 시점 재무로 얹으려면 두 가지가 성립해야 한다:
1. **look-ahead 방어** — 그 시점에 실제 공시돼 있던 재무만 사용.
2. **생존편향 방어** — 그때 상장돼 있었지만 이후 폐지된 종목의 과거 재무도 조회 가능.

이 노트는 실제 DART로 위 둘을 검증한 기록이다. **수집기를 짓기 전에** 확인한 이유는
생존편향 방어가 "폐지 종목 재무 조회 가능 여부"에 통째로 걸려 있기 때문.

## 1. look-ahead — 공시일(PIT) 확보 가능? → ✅ 가능

- DART 재무 응답(fnlttSinglAcntAll)의 `rcept_no`(접수번호) **앞 8자리 = 실제 공시일**.
  예: 삼성 FY2024 사업보고서 `rcept_no=20250311001085` → 2025-03-11 공시.
- 공시검색 API(list.json)도 `rcept_dt`로 직접 제공. 연간보고서는 **다음 해 3월 초~4월 초** 공시(매년 상이).
- **결론**: 보수적 시차를 추정할 필요 없이 **실제 공시일**을 쓴다.
  PIT 규칙 = 리밸런싱일 D에서 `공시일 ≤ D` 인 가장 최근 보고서만 사용.

## 2. 생존편향 — 폐지 종목 과거 재무 조회 가능? → ✅ 가능 (비금융)

**방법**: KRX 2016-06-30 단면(2,011종목) − 2025-12-30 단면(2,738종목) = 그새 폐지 239종목.
그중 DART corp_code 매핑 215/239(90%). 표본 15개의 2015 사업보고서 실조회.

**결과**: 완전 10 / 부분 1 / 없음 4

| 구분 | 종목 | 해석 |
|------|------|------|
| 완전(10) | 디엘건설·롯데푸드·쌍용씨앤이·동성화학 등 | 재무+자본총계+공시일 정상 → PIT F-Score 계산 가능 |
| 부분(1) | 고려개발 | 재무는 오나 '자본총계' 계정명 매칭 실패(옛 표기차) |
| 없음(4) | 우리은행·메리츠화재·KB손해보험·KB증권 | **전부 금융사** — F-Score 원래 제외 대상. 손실 아님 |

## 3. 결론

- **폐지된 일반기업의 상장 시절 재무는 공시일까지 붙어 정상 조회됨 → 생존편향 방어 가능.**
- "없음"은 전부 금융사(은행·보험·증권)로, F-Score가 원래 배제하므로 실질 손실 없음.
- 즉 **비금융 유니버스에서 PIT + 생존편향이 모두 성립**.

## 4. 수집기 설계에 반영할 것

1. **유니버스**: 매 리밸런싱일 KRX 단면(그 시점 상장 종목) → 생존편향 안전.
2. **매핑**: 종목코드→corp_code (CORPCODE). 커버리지 ~90%. 미매핑·지주사 재편(예: 현대증권→KB증권 corp_code 이동)은 소수 → 로그로 남기고 스킵.
3. **금융사 제외**: 은행·보험·증권·리츠는 F-Score 부적용 → 유니버스에서 배제(Piotroski 표준).
4. **계정명 별칭 보강**: 옛 공시 표기 차이(고려개발 사례) 대응 위해 metrics 별칭 확장.
5. **콜 수 절감**: 가치(PBR+PER 랭킹)로 싼 상위 N만 F-Score 계산 → 가치→품질 2단.
6. **정정공시**: v1은 최종 수치+원공시일 사용(미세 look-ahead 감수). 나중 정밀화.

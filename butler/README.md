**2008급 근사(코스피 기반, 보수적)**: 방어 -25% · 균형 -35% · 성장 -42% (실제는 채권 완충으로 덜).

- **균형형 확정**: MDD -21%(현대)/~-35%(2008급), 순수 코스피(-47%) 대비 **절반**. 정적 분산으로
  목표 C(시장수익 유지+MDD 감소)를 매매·whipsaw 없이 달성. → policy `_measured_mdd` 갱신.
- **데이터 한계(정직)**: 4자산 실데이터 2011~ → 2008은 코스피 근사(글로벌≈국내·채권≈현금 보수적).
- **헤지/언헤지 방향**: 시뮬은 헤지형 글로벌(장기데이터 제약) → 실투자 언헤지는 위기 때 원화약세
  자연헤지로 **덜 빠짐** → **시뮬 낙폭은 보수적(약간 나쁘게 나온) 추정.** 실제는 이보다 나을 것.

## II. 가변 적립 배분 엔진 (allocate.py) ✅  — v0.3 policy(6:2:2 고배당 확대판)
자산 6개: 국내코어 21·글로벌코어 21·고배당ETF 14·개별주(위성) 14·채권 20·현금 10(%).
입력(이번 금액 + 자산별 잔고) → policy 비율로 배분 지시. **알고리즘(결정론):**
1. 후잔고 목표액 = (현잔고+입력) × 목표비중. 위성 목표 = min(비중, 상한 14%).
2. 입력액 ≤ 총미달: 미달에 **비례 배분**(목표미달 자산 우선, 초과 자산 0).
3. 입력액 > 총미달: 미달 다 채우고 나머지는 **위성 뺀** 비중대로(위성 상한 방어).
4. 계좌: 현금→유동성버퍼(CMA), ETF→연금(ETF만)→ISA→일반, **개별주→ISA→일반(연금 불가)**.
5. 리밸: 목표 ±5%p 초과 자산 플래그. 위성 상한 초과 시 "코어/고배당ETF로 재배분" 경고.

검산(예시 1200만, 잔고 0): 국내코어·글로벌 각 252만, 고배당·위성 각 168만, 채권 240·현금 120만.
**주식 내부 코어60:고배당20:위성20 정확.** 계좌: 현금 CMA, ETF 912만→연금600·ISA232·일반80,
위성 168만→ISA(연금 못 담음). ✓

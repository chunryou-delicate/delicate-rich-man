# 진심 v0.5 착수 지시 (실전 트랙 = 집사)

> 연구 트랙(한국 종결/미장)과 **별개**. 실제로 굴릴 도구 v0.5.
> **주문·API·키 없음** — 읽고 손입력 → 지시서 출력이 전부. 완성 즉시 실사용 가능 범위.
> 유일한 정책 = [policy.json](policy.json). 엔진은 그 밖의 판단을 하지 않는다.

## 구현 범위 (v0.5)

### 1. 낙폭 시뮬 먼저 (프리셋 실측 검증) ← 첫 작업
- 3국면 지수 데이터(코스피 / 글로벌 대용) 재사용해, 세 프리셋의 **실제 과거 MDD·회복기간** 표로:
  - 방어 50/35/15 (주/채/현) · **균형 70/20/10** · 성장 85/10/5, 주식 내부 국내:글로벌 50:50.
- 목적: 발주낸 추정값(-20/-30/-40)이 아니라 **실측 낙폭**을 보고 균형형 확정.
- ⚠ 글로벌(S&P500)·채권 데이터 짧으면 국내 지수/가용범위로 근사하고 **한계 명시**.

### 2. 가변 적립 배분 엔진 (핵심)
- 입력: 이번에 넣을 금액(가변) + 현재 잔고(자산별, 손입력).
- 출력: policy 비율(주70/채20/현10, 주식 국내:글로벌 50:50)에 맞추도록 **이번 입력액을 어디에 얼마씩**
  배분하는 지시. 목표비율 미달 자산 우선 채움.
- 계좌 우선(policy.accounts.priority_fill): 연금→ISA→일반. 한도는 policy에서 읽되 **미입력이면 물음(하드코딩 금지)**.
- 정기/수시 겸용: "이번에 얼마" 한 번 물으면 됨.

### 3. 리밸런싱 체크
- 현재 배분이 목표에서 ±5%p 벗어난 자산 있으면 지시에 포함, 없으면 "리밸런싱 불필요".
- **매도 지시 자동생성 금지(firewall)** — 리밸런싱 내 부분매도만, 그것도 밴드 초과 시.

### 4. 지시서 출력 (단일 파일 HTML)
- 예: "이번 300만 → 연금 KODEX200 X주 / ISA TIGER미국S&P500 Y주 / 일반 채권ETF Z주 / CMA 현금 W만.
  리밸런싱 불필요. 끝."
- 대시보드 겸용: 현재 vs 목표 배분 막대, 계좌별 잔고, 다음 액션. 단일 HTML.
- **주문 버튼 없음.** 사용자가 지시서 보고 증권사에서 손으로 실행(A안).

## 엔진 규율 (코드 레벨)
- 100% 결정론. policy.json + 입력값만으로 계산. **예측·전략변경 로직 없음.**
- AI 판단 로직 엔진에 안 넣음. 뉴스·센티 입력 없음(v0.5엔 정보 레이어 없음).
- policy 변경은 파일 수정 + 30일 숙려(주석 개정일 기록). **코드가 policy 자동수정 금지.**

## 안 하는 것 (v0.5 경계)
- 주문·체결·API·키 일절 없음. 잔고도 손입력(v1에서 KIS 조회연동).
- 완전자동(B안)은 v2. 지금 주문키 발급 금지.
- 정보 레이어는 v1 이후.

## 순서
낙폭 시뮬(1) → 균형형 확정 → 배분엔진(2) → 리밸체크(3) → 지시서 HTML(4).
